## 摘要
科学机器学习通常需要将已知物理规律与从数据中学习的未知参数或修正项相结合。现有方法要么忽略已知结构，将其编码为软罚项，要么需要为每个方程手动编写PyTorch代码。我们提出了神经编译器（The Neural Compiler），该系统可将用类一阶Scheme表达式语言编写的程序翻译成冻结且可微分的PyTorch模块。这些模块能匹配源程序至浮点精度，并通过自动求导提供梯度。在混合模型中，编译后的模块精确编码已知物理规律，而学习组件则对未知余项进行建模。我们针对六个实验领域评估了该编译器：费曼物理方程、洛特卡-沃尔泰拉动力学、阻尼摆、一维热方程、三维矢量力学以及组合泛化。对于单个方程，编译后的模块在数值上匹配手写PyTorch实现，无编译精度损失。仅用1到4个可训练参数，编译后的模型在多数情况下恢复物理常数的误差小于1%，而拥有超过8500个参数的标准PINN基准模型则显示出7%到93%的误差。编译后的模块还能以零误差进行组合，而神经网络近似在深层组合链中可能累积较大误差。该编译器的主要价值不在于比手写方程提升精度，而在于系统性可组合性：它能够从符号规范生成正确、可微分的模块，无需为每个方程手动重写。系统支持51种基本操作，包括向量和矩阵代数，从而支持PDE离散化和混合科学模型。这种“字符串输入、模块输出”的接口也为大语言模型提供了天然目标，可将科学描述翻译为可执行的可微分模块。

## Abstract
Scientific machine learning often requires combining known physics with unknown parameters or correction terms learned from data. Existing approaches either ignore known structure, encode it as a soft penalty, or require hand-written PyTorch code for each equation. We present The Neural Compiler, a system that translates programs written in a first-order Scheme-like expression language into frozen, differentiable PyTorch modules. These modules match the source program to floating-point precision and provide gradients through autograd. In hybrid models, the compiled module encodes known physics exactly while learned components model the unknown remainder. We evaluate the compiler across six experiment domains: Feynman physics equations, Lotka-Volterra dynamics, a damped pendulum, a one-dimensional heat equation, three-dimensional vector mechanics, and compositional generalization. Compiled modules match hand-coded PyTorch implementations numerically for single equations, showing no accuracy loss from compilation. With only 1 to 4 trainable parameters, compiled models recover physical constants to less than 1 percent error in most cases, while standard PINN baselines with more than 8500 parameters show 7 to 93 percent error. Compiled modules also compose with zero error, while neural approximations can accumulate large errors in deep composition chains. The main value of the compiler is not improved accuracy over hand-coded equations, but systematic composability: it generates correct, differentiable modules from symbolic specifications without rewriting each equation by hand. The system supports 51 primitive operations, including vector and matrix algebra, enabling PDE discretizations and hybrid scientific models. This string-in, module-out interface also provides a natural target for large language models that translate scientific descriptions into executable differentiable modules.

## 1. 论文的核心问题与整体含义（研究动机和背景）

科学机器学习领域面临一个核心矛盾：研究者通常拥有部分已知的物理领域知识（如已知方程形式、特定定律），但又需要从数据中学习未知参数或修正项。处理此类融合问题，现有方法存在三个令人不满意的选项：

- **选项一（神经网络/Neural ODE）**：完全丢弃已知物理结构，让神经网络从头学习全部映射。这需要大量参数，且学到的是不可解释的黑箱权重。
- **选项二（物理信息神经网络PINNs）**：将已知物理方程作为软罚项加入到损失函数中。模型被允许违反物理规律，这导致了软约束的不稳定性 —— 模型可能为了拟合数据而放弃物理正确性，尤其在外推时表现脆弱。
- **选项三（手写代码）**：为每个方程手动编写可微分的PyTorch模块。这虽然精确，但需要每遇到一个新方程就重新实现，且不同模块之间难以自动组合。

因此，论文要回答的核心问题是：**能否开发一个系统，能将符号形式的已知物理知识自动、精确地转换为可微分计算模块，既保留硬约束的准确性优势，又避免重复实现的工作？**

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想：编译而非近似

提出**第四种选项**：将已知物理知识从符号规格说明自动编译成可微分的PyTorch模块。编译后的模块在被使用时是**冻结的**（不包含可训练参数），仅负责精确计算已知的物理部分；而未知部分则由附加的可训练神经网络（MLP）来学习。这种混合架构保证了：

- 编译模块在其安全域内与源程序输出完全一致，无近似误差。
- 提供通过autograd获得的精确梯度。
- 模块之间组合时误差为零。
- 迁移和组合仅需修改源字符串，无需重写代码。

### 2.2 关键技术细节

- **源语言 $L$**：使用Scheme语法的一阶纯算术表达式语言。支持51种基本操作，包括标量算术（四则运算、三角函数、指数、对数等）、向量运算（点积、叉积、范数、归一化等）、矩阵运算（乘法、行列式、求逆、外积等）以及控制流（let绑定、尾递归循环、条件分支）。
- **编译流水线**：四个阶段 ——
  1. **Parse**：递归下降解析S-表达式为抽象语法树（AST）。
  2. **ANF Transform**：将AST转换为A范式（A-Normal Form），其中所有操作参数必须是常数或变量。含复合子表达式的部分通过let绑定引入临时变量。
  3. **Tail-Call Optimization**：将尾递归函数转换为迭代循环，提升效率。
  4. **Graph Construction & Compilation**：每个ANF let绑定成为有向无环图（ComputeGraph）的一个节点，按拓扑顺序编译为指令序列，生成DirectModule（一个PyTorch `nn.Module`）。
- **理论保证**：论文给出了四个定理和推论 ——
  - **定理2（编译正确性）**：对于任意程序 $P \in L$ 和输入 $x \in D_P$（安全域），编译后的模块 $M$ 满足 $M(x) = [\![P]\!](x)$，即两者在浮点精度下完全一致。
  - **定理3（梯度正确性）**：在可微区域，梯度和源程序梯度一致。
  - **推论4（梯度流经冻结子图）**：混合模型中梯度可经过冻结的编译模块传递到可训练部分。
  - **定理5（编译组件精确性）**：编译模块在安全域上贡献零近似误差，所有误差来自学习组件。
  - **命题6（组合误差界）**：编译模块链的组合误差为零；而神经网络近似链的误差会因下游利普希茨常数而指数级放大。
- **三种混合架构模式**：
  1. **已知结构 + 未知参数**：整个方程已知，常数为可训练参数。
  2. **已知项 + 学习修正**：编译模块处理已知项，MLP学习余项，输出相加。
  3. **组合流水线**：多个编译模块串行链接，零误差。

1. **编译与手写代码数值一致**：在单一方程上编译模块与手写PyTorch代码的输出在机器精度下完全相同（表1–4）。
2. **硬约束显著优于软约束**：编译/手写模型（硬约束）在参数恢复上表现远优于PINN（软约束）—— 热方程 $\alpha$ 恢复误差0.0% vs. 92.7%，阻尼摆 $g/L$ 恢复误差0.08% vs. 41.1%（图10）。
3. **参数效率极高**：1–4个参数即可恢复物理常数至<1%误差（费曼方程、洛特卡-沃尔泰拉等），而MLP需数千参数仍在外推上失效（表1、2）。
4. **精确组合**：编译模块链的组合误差为零；而神经网络近似链在深度和线性放大下误差可爆炸至$5.9 \times 10^{9}$（表5）。
5. **主要价值在于系统化可组合性**：编译器能为任何支持语言内的表达式自动生成可微分模块，无需重复手写代码，且可被LLM调用作为文本到模块转换的管道。

## 7. 优点

- **理论保证清晰**：给出了编译正确性、梯度正确性、组合误差界等形式化定理，确保了方法的可靠性。
- **精确性突出**：在混合模型中，编译部分对已知物理做精确计算，没有近似误差，这一点明显优于PINN。
- **低计算开销**：编译一次性耗时不到150微秒，批量推理吞吐量高，实际训练开销仅增加2.5–3.8倍，易于集成。
- **支持向量/矩阵运算**：能处理PDE离散化等复杂表达式，拓展了应用范围。
- **接口自然**：字符串输入、模块输出，天然适合与LLM交互，开辟了“自然语言到可微分物理模块”的路径。
- **实验设计系统化**：覆盖代数、ODE、PDE、矢量运算和组合泛化等关键场景，验证了核心主张。

## 8. 不足与局限

- **源语言受限**：支持51种操作，但缺少高阶函数、字符串处理、通用数据结构（如列表、字典），虽覆盖广泛科学公式，但并非万能。
- **需已知结构**：编译器只是利用已知结构，无法发现未知的结构。当方程形式完全未知时需依赖符号回归等互补方法。
- **混合模型中的信用分配问题**：当编译组件和学习组件功能形式重叠时（如阻尼摆场景2），优化器可能将功能在两者间分配不当，导致可解释性下降（文中3.3节）。
- **训练开销**：相对于纯MLP，混合模型训练慢2.5–3.8倍，对于简单表达式来说，编译器的工程便利性价值可能掩盖计算开销。
- **部分方程优化失败**：15个费曼方程中，谐波振荡器（周期性损失景观）和洛伦兹因子（$v\to c$ 奇异性）的常数恢复失败，表明梯度下降可能陷入陷阱。
- **未提供大规模或真实数据实验**：所有实验均在合成数据上进行，未涉及真实物理观测（如实验测量、噪声结构复杂），也未提及与真实传感噪声、模型误差的交互。
