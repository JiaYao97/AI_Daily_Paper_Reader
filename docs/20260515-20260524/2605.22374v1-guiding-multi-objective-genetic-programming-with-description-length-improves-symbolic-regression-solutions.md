## 摘要
基于遗传编程的符号回归（GPSR）可能遭受过拟合和结构膨胀，尤其在存在噪声时。本文评估了描述长度（DL）和分数贝叶斯因子（FBF）准则，作为原则性、数据高效的启发式替代方法，用于选择泛化良好的紧凑表达式。我们使用基于Fisher信息的参数编码实现DL，并在多个数据集（包括含噪合成基准和真实回归问题）上将其与AIC和BIC比较。我们研究了三种搜索/选择策略：（i）多目标搜索精度和程序长度，随后进行DL/FBF选择；（ii）直接将DL作为目标的多目标搜索；以及（iii）以DL/FBF作为适应度的单目标优化。跨数据集发现，与AIC/BIC基线相比，DL/FBF事后选择提高了测试性能，且BIC结合来自DL/FBF的相同函数复杂度惩罚产生类似结果。相反，在单目标GPSR中直接使用DL/FBF作为适应度函数常导致过早收敛到过于简单的模型。最后，我们给出了在遗传编程工作流中使用DL/FBF作为鲁棒模型选择工具的实用指南。

## Abstract
Symbolic regression with genetic programming (GPSR) may suffer from overfitting and structural bloat, especially when noise is present. In this paper we evaluate description length (DL) and fractional Bayes factor (FBF) criteria as principled, data-efficient alternatives to heuristics for selecting compact expressions that generalise well. We implement DL using a Fisher-information-based parameter encoding and compare it to AIC and BIC across multiple datasets, including noisy synthetic benchmarks and real-world regression problems. We study three search/selection strategies: (i) multi-objective search for accuracy and program length followed by DL/FBF selection; (ii) multi-objective search using DL directly as an objective; and (iii) single-objective optimisation with DL/FBF as the fitness. Across datasets we find that DL/FBF post-selection improves test performance compared to AIC/BIC baseline and that BIC in combination with the same function complexity penalty from DL/FBF produces similar results. In contrast, using DL/FBF directly as a fitness function in single-objective GPSR frequently induces premature convergence to overly simple models. We conclude with practical guidance for using DL/FBF as robust model-selection tools in genetic programming workflows.

### 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：基于**香农信息论**和**柯尔莫哥洛夫复杂度**的MDL原则，选择能够以**最短总长度**描述数据的模型（模型描述长度 + 数据在模型下的编码长度）。一个好的模型应该既准确又简洁。FBF是贝叶斯证据的一种近似，用于模型比较。
- **关键技术细节**：
    1.  **模型选择准则**：
        - **AIC / BIC (基线)**：
            $$AIC = -2 \log L + 2p$$
            $$BIC = -2 \log L + p \log m$$
            其中 $L$ 是似然，$p$ 是参数数量，$m$ 是样本数。
        - **描述长度（DL）**：包含数据似然、函数复杂度和参数复杂度。
            $$
            DL = \underbrace{-\log L}_{\text{数据似然}} + \underbrace{k \log n + \sum_{c_i} \log c_i}_{\text{函数复杂度}} + \underbrace{\frac{1}{2} \sum_{i=1}^p \max(0, \log S_{ii} - \log 3 + \log |\hat{\theta}^{rot}_i|)}_{\text{参数复杂度}}
            $$
            - **函数复杂度**：$k \log n$ 中，$k$ 是表达式树节点数，$n$ 是不同符号（运算符、变量）数量。$\sum \log c_i$ 是常数编码长度。
            - **参数复杂度**：这是关键创新。它通过**奇异值分解（SVD）** 计算观测Fisher信息矩阵（FIM），在新的旋转参数空间（$\hat{\theta}^{rot}$）中量化参数刚度。**“刚度”** 参数（$S_{ii}$ 很大，微小改变即显著影响预测）需要更精确的编码，复杂度高；**“松散”** 参数（$S_{ii}$ 很小）可以被压缩。这比仅使用FIM对角线的方法更能处理参数之间的强相关性。
        - **分数贝叶斯因子（FBF） & BIC_SR**：
            $$FBF = -(1-b) \log L + \text{func comp} + \frac{p}{2}(\log(2\pi e^{1-\log 3}) - \log b)$$
            $$BIC_{SR} = -2 \log L + 2 \cdot \text{func comp} + p \log n$$
            这两个准则都采用了与DL相同的函数复杂度惩罚项（`func comp`），但在参数复杂度处理上更简单（FBF使用分段数据校准，BIC_SR忽略参数刚度）。它们计算成本更低。
    2.  **参数优化**：所有模型的数值参数 $\theta$ 和噪声方差 $\sigma^2_{err}$ 通过**Levenberg-Marquardt算法**进行非线性最小二乘优化（记忆优化）。
- **算法流程**：
    - **多目标（MO-Length）**：
        1.  使用NSGA-II算法，同时优化两个目标：**负对数似然（NLL）** 和**程序长度**。
        2.  每一代，从当前Pareto前沿中，使用一个模型选择准则（AIC/BIC/BIC_SR/DL/FBF）**选择**一个最佳模型。
        3.  最终返回选出的模型。
    - **多目标（MO-DL）**：
        1.  与MO-Length类似，但第二个目标改为直接优化**DL惩罚项**（函数复杂度+参数复杂度）。
        2.  最终模型通过最小化DL从前沿中选出。
    - **单目标（SO）**：
        1.  直接将AIC、BIC、DL等准则作为适应度函数进行单目标优化。
        2.  算法目标是最小化这个单一的混合准则。

### 6. 主要结论与发现

1.  **DL/FBF/BIC_SR作为后选择工具效果卓越**：在**多目标（MO-Length）** 框架下，使用DL/FBF/BIC_SR从Pareto前沿中选择模型，其测试性能**显著优于**使用AIC或BIC进行选择。
    - 它们能有效控制过拟合和膨胀，选出更短、更泛化的表达式。
    - 在多个基准测试上，**BIC_SR** 和 **FBF** 在性能上与DL非常接近，但计算成本更低（`P(DL > FBF) = 51.7%`，在等价区间内）。
2.  **缺乏函数复杂度惩罚的准则会失败**：AIC和BIC因其惩罚项过于简单，在GPSR中表现出明显的过拟合。
3.  **单目标直接优化DL/FBF不可取**：将DL/FBF直接作为**单目标（SO）** 的适应度函数，会由于过强的复杂度压力，导致种群**过早收敛**到过于简单、欠拟合的表达式，性能明显逊于MO-Length框架。
4.  **DL作为直接优化目标（MO-DL）无显著优势**：在MO中直接用DL惩罚项替代程序长度，虽然能找到DL值更低的表达式，但其**测试误差与MO-Length+DL后选择相当**，且计算成本更高。
5.  **MDL能自适应调节模型复杂度**：MDL选择能够根据噪声水平和样本量自动调整模型大小。高噪声/小样本时选择更短模型，低噪声/大样本时选择更长模型。
6.  **实用建议**：推荐使用**多目标优化（NLL和长度）**，然后使用**DL、BIC_SR或FBF**进行事后模型选择。**不推荐**在单目标搜索中直接使用这些准则。

- **算法实现精细**：DL实现中采用SVD处理参数相关性，解决了GPSR中常见参数高度相关的问题，这是一个重要的创新点。
- **消融实验设计巧妙**：通过设计MO-Length, MO-DL, SO三种策略，**有效分离了“搜索”和“选择”两个环节**，能够清晰判断DL/FBF在不同环节的作用和局限性，结论非常深刻。
- **实验分析深入**：不仅报告平均性能，还通过大量图表深入分析了程序长度、Pareto前沿、参数刚度、外推行为等，研究非常细致。统计比较方法（BBT模型）先进且规范。
- **实用价值高**：结论清晰，提供了可直接执行的实用建议（使用MO搜索 + DL/FBF/BIC_SR后选择），并明确指出应避免的做法（单目标直接优化DL）。

### 8. 不足与局限

- **似然函数假设**：所有实验均基于**高斯似然**（i.i.d. 高斯噪声），实际应用中的数据可能不满足此假设。对于分类或其他问题，可能需要不同的似然函数。
- **函数复杂度惩罚的通用性**：论文采用的 `k log n` 形式的函数复杂度惩罚虽然整体有效，但最优惩罚强度可能因问题而异。文中提到太小或太大的惩罚可能对特定数据集效果更好，如何自适应地确定惩罚是一个开放问题。
- **计算成本**：虽然DL的计算成本低于参数优化，但其SVD分解的$O(p^3)$复杂度在处理参数数量很多的模型时可能成为瓶颈。
- **未与其他先进方法对比**：实验基线是AIC/BIC，但未与领域内其他复杂的模型选择方法（如**结构风险最小化（SRM）** 或**PAC-Bayes界**）进行对比。作者在讨论部分承认了这一点，并将其列为未来工作。
- **实验偏差风险**：
    - **SRBench++数据集**：作者指出，该基准中的许多小数据集上，几乎所有复杂模型都过拟合，这可能导致对MDL等“强约束”准则的测试不够区分度，可能高估了其相对某些更强搜索算法的能力。
    - **Tower数据集**：该数据集上AIC表现最佳，作者分析可能是因为其训练/测试划分破坏了i.i.d.假设，导致基于似然的准则失效。这体现了基于似然的准则的局限性。
