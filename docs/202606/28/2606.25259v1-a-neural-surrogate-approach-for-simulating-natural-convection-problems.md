## Abstract
This paper presents a neural surrogate approach for improving the accuracy of natural convection problems simulated with a Boussinesq flow model (incompressible flow with heat transfer). Our approach, based on Fourier neural operators, uses training data consisting of matched pairs of simulations run under the computationally cheaper yet less accurate Boussinesq flow model and a more computationally expensive and more accurate compressible flow model. In both cases, we implement our parallelized simulation codes based on an implicit monolithic mixed finite element method (FEM) approach using the open-source FEniCSx framework. Our implementations are validated against a commercial software package, COMSOL, as well as standard test problems from the literature. We include a careful discussion and analysis of data set generation and present learning results in two and three spatial dimensions. Using compressible flow results as high-fidelity reference solutions, our learning approach, with a single model evaluation per simulation, substantially improves the per-channel accuracy of Boussinesq predictions, with structural similarity (SSIM) close to unity across all flow variables and test distributions and corresponding mean-squared error reductions of one to nearly three orders of magnitude. All code and data is released as open-source.

- **数据生成硬件**：可压缩流求解器使用多核节点（强缩放实验最高 64 核），时间在 2D 上约 159 秒/样本（64×64 网格），3D 上约 3246 秒/样本（24³ 网格）。
- **训练时间**：论文未明确报告训练耗费的墙钟时间，但提到使用 AdamW 优化器，2D 学习率 $10^{-4}$，3D 学习率 $5\times10^{-4}$，余弦退火调度+10 轮线性预热的调度，权重衰减分别为 $10^{-3}$ 和 $5\times10^{-3}$。2D 批次大小 64，3D 批次大小 32，训练 200 轮。总训练时长取决于 GPU 和数据集大小，未量化提供。
- **推理时间**：2D FNO 推理 1.1 ms/样本，3D FNO 推理 1.6 ms/样本，远小于 Boussinesq 求解时间（2D 约 11 s，3D 约 584 s）和可压缩流求解时间（2D 约 159 s，3D 约 3246 s）。

## 5. 实验数量与充分性

- **数量**：
  - 求解器验证：网格收敛（5 套网格×2 种公式）、时间收敛（两种格式）、COMSOL 对比（5-6 个 Rayleigh 数×2D 和 3D）、Nusselt 数对比、保守性分析（质量、能量、熵）、制造解验证。
  - 数据集分析：对 500 个随机样本做 PCA/UMAP 分析，展示分布覆盖。
  - 代理模型实验：2D 中 2 种训练集 × 3 种测试集 = 6 个配置，每配置 200 样本的逐通道 MSE 和 SSIM 报告。3D 中 1 种训练集 × 3 种测试集 = 3 个配置，每配置 100 样本的逐通道 MSE 和 SSIM。另外还有 FNO 与 U-Net 的对比（在随机测试集上）。
- **充分性与公平性**：实验覆盖了网格、时间收敛、多种 Rayleigh 数、高低精度求解器验证、多分布测试、跨数据推广测试、OOD 测试，并采用了标准测试用例（方腔、制造解）。对比基线（U-Net）使用完全相同的训练协议（优化器、调度、批次、轮次），保证了公平性。但缺乏与其他神经算子（如 DeepONet、Graph Neural Operator）或较新的扩散模型的对比；缺少不同随机种子/多次重复实验的统计波动报告（仅报告单次平均指标）。3D 数据较小（仅 1000 训练），可能限制泛化结论的稳定性。

## 6. 论文的主要结论与发现

- **求解器有效**：所开发的隐式整体混合有限元求解器实现正确，与 COMSOL 和文献基准在温度廓线、Nusselt 数上高度一致，并满足质量守恒到机器精度，能量和熵平衡在一阶隐式格式精度范围内成立。
- **数据集生成策略有效**：随机边界条件生成器产生的训练数据在 PCA/UMAP 嵌入中连续分布、无空洞，确保训练代理模型能够覆盖广泛的流动模式。
- **神经代理显著提升精度**：训练后的 FNO 在分布内测试集上的 SSIM 均接近 1（≥0.995），MSE 较原始 Boussinesq 降低 1–3 个数量级。即使在常数边界条件（Old）或 OOD 测试集上，随机训练的 FNO 仍保持低误差（2D SSIM ≥ 0.994，3D SSIM ≥ 0.876），而常数边界条件训练的 FNO 在随机测试集上误差反而放大。
- **架构选择重要但数据分布影响更大**：FNO 在参数相当的紧凑版本下仍优于 U-Net（MSE 降低 32–65% in 2D, 80–98% in 3D），但跨数据推广实验表明，训练数据分布的选择（随机 vs 常数）对泛化能力的决定性远大于架构差异。
- **实际成本效益**：以 4000 个 2D 训练样本为例，训练成本一次支付，每次新模拟仅需做一次 Boussinesq 求解 + 1.1 ms 推理，替代完整可压缩流求解；在 2D 中约 4300 次评估后收回训练投入，在 3D 中约 1200 次评估后收回。

## 7. 优点

- **开源生态**：全部代码和数据集均开源，基于成熟框架 FEniCSx 和 PyTorch，便于复现与扩展。
- **完整的求解器验证**：不仅与商业软件 COMSOL 对比，还进行了网格/时间收敛、制造解验证、热力学第一第二定律的平衡分析，验证体系全面扎实。
- **高质量数据生成策略**：随机傅里叶级数和 Voronoi 方法在理论上保证了样本光滑、有界、可测，并自然包含常数边界条件作为特例，避免了人为截断或投影。
- **严格的跨分布推广实验**：通过设计“常数训练集→随机测试集”、“随机训练集→OOD 测试集”等不对称实验，清晰证明了随机数据生成对泛化能力的决定性作用，避免了模型选择偏差。
- **清晰的应用场景定位**：论文明确讨论了代理模型的收支平衡点，并指出其最适合在固定几何和网格下需要大量模拟的应用（不确定性量化、参数扫描等）。

## 8. 不足与局限

- **网格与几何固定**：当前 FNO 代理模型仅在单一分辨率（2D 64×64，3D 24³）和固定方形/立方体腔体上训练与评估，不适用于其他几何形状或更粗/更细网格，缺少网格自适应或多分辨率适配能力。
- **流态覆盖有限**：测试的 Rayleigh 数上限为 $10^6$，Mach 数极低（<0.002），未覆盖湍流或其他高马赫数可压缩效应。论文也明确未引入 RANS/LES 等湍流模型。
- **3D 训练数据偏少**：仅 1000 个 3D 训练样本，可能影响模型在高维参数空间中的泛化稳健性；OOD 测试集中 MSE 增大约 5–10 倍，部分通道 SSIM 降至 0.88，提示数据量尚不足。
- **统计报告不完整**：所有指标仅报告单次训练的一个测试平均值，未报告多次重复实验的均值与方差（由于训练成本高昂而来报告），也无法判断模型随机性对结果的影响程度。
- **缺乏与其他神经算子架构的对比**：对比仅针对 U-Net，未与 DeepONet、Graph Neural Operator、Transformer-based operator 等 SOTA 方法比较，结论的普适性受限。
- **时间成本披露部分模糊**：训练时长未明确给出；3D 求解器计时仅取 N=5 次可压缩流运行的平均值，样本量较小；推理时间仅含 GPU 前向时间，未包含数据预处理/后处理时间。
- **未探索主动学习或在线学习**：论文对数据生成进行了详细讨论但未尝试利用代理模型的不确定性进行主动选择或在线细化。
