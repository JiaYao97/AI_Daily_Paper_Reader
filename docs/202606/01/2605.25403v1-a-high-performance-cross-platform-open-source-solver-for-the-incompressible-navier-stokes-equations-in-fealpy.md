## 摘要
为应对异构硬件上的性能可移植性以及传统计算流体动力学（CFD）软件高使用门槛的双重挑战，本文介绍了FEALPy.CFD——一个在FEALPy框架内开发的高性能、开源不可压Navier-Stokes方程求解器。该求解器的核心创新在于其后端无关设计，支持NumPy、PyTorch和JAX等多种计算后端，通过最少的代码修改实现CPU和GPU计算之间的无缝切换，从而最大化硬件利用率和代码可移植性。其高度模块化的架构为各种空间离散化方案提供了可组合组件库，极大简化了新算法的开发和集成。在基准测试案例上的验证表明，所实现的数值格式达到了理论收敛阶数。此外，针对不同计算任务选择合适后端架构的能力充分释放了硬件潜力，带来了显著的效率提升。通过降低高性能、跨平台流体动力学模拟的技术门槛，FEALPy.CFD为学术研究、工程应用和可复现计算科学提供了一个强大且易用的工具。

## Abstract
To address the dual challenges of performance portability across heterogeneous hardware and the high usability barriers of conventional computational fluid dynamics (CFD) software, this paper introduces FEALPy.CFD, a high performance, open-source solver for the incompressible Navier-Stokes equations developed within the FEALPy framework. The solver's core innovation is its backend-agnostic design, which supports multiple computational backends like NumPy, PyTorch, and JAX to enable seamless switching between CPU and GPU computations with minimal code modification, thereby maximizing hardware utilization and code portability. Its highly modular architecture provides a library of composable components for various spatial discretization schemes, greatly simplifying the development and integration of new algorithms.Validation on benchmark cases confirms that the implemented numerical schemes achieve their theoretical orders of convergence. Furthermore, the capability to select a suitable backend architecture for different computational tasks fully leverages the hardware's potential, delivering substantial efficiency gains. By lowering the technical barrier to high-performance, cross-platform fluid dynamics simulation, FEALPy.CFD offers a powerful and accessible tool for academic research, engineering applications, and reproducible computational science.

### 方法论：核心思想与关键技术

FEALPy.CFD的核心思想是**解耦算法实现与硬件执行**，其技术细节如下：

- **后端无关设计 (Backend-Agnostic Design)**：
    - **思想**：通过一个统一的抽象层（Tensor Level）封装底层张量计算库的差异。
    - **实现**：支持`NumPy`（CPU原型开发）、`PyTorch`（CPU/GPU大规模计算）、`JAX`（JIT编译加速）等计算后端。用户只需一行代码即可切换后端（`bm.set_backend('pytorch')`），无需修改主算法代码。

- **高度模块化架构**：
    - **思想**：将物理问题描述、数值方法、求解流程和公共功能进行解耦。
    - **实现**：架构围绕四个核心模块设计：
        - **`MathModel`**：问题定义中心，包含内置基准案例库和用户自定义问题的标准化接口（支持Sympy符号推导）。
        - **`Equation`**：物理模型解析模块，负责将数学模型转化为可计算对象，管理物理系数和边界条件。
        - **`Simulation`**：计算核心，负责空间离散化（FEM, FVM, SPH）和时间离散化（BDF等）。采用“过程分解”设计（例如将IPCS算法分解为动量预测、压力泊松方程、速度修正等独立步骤），提供细粒度的编程接口，方便研究人员替换或改进特定环节。
        - **`Common`**：公共功能模块，提供数据I/O（VTK格式兼容）、快速可视化（Matplotlib）等功能。

- **支持的算法与公式**：
    - **控制方程**：支持Stokes方程、稳态/非稳态不可压Navier-Stokes方程，并内置了广义牛顿流体模型（如Power-Law、Cross-WLF模型）以处理非牛顿流体。
    - **线性化与解耦**：实现了Oseen迭代和Newton迭代处理非线性项，以及经典的Chorin投影法和增量压力校正格式（IPCS）解耦速度-压力。

    - **SPH方法**：集成了多种核函数（B-spline, Wendland）以及一系列修正算法，包括：固体壁面边界处理（虚拟粒子法）、核梯度修正、人工粘性与低耗散Riemann求解器、粒子位移技术等。

### 优点

1.  **创新的后端无关设计**：这是本文最核心的亮点。它将硬件优化的复杂性从算法开发者转移到框架层面，极大提升了代码的可移植性和未来硬件扩展的潜力。

### 不足与局限

1.  **工程应用验证有限**：论文的“高性能”结论主要来自对特定算例（圆柱绕流、Taylor-Green涡旋）的速度测量，**缺乏与OpenFOAM、FEniCS等主流工业级软件在大规模、复杂工程问题上的直接性能和精度对比**。这使得其性能声明缺少一个关键的行业基准。
